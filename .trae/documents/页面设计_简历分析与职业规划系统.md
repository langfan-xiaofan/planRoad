# 页面设计说明（Desktop-first）

## 全局设计（适用于所有页面）

### Layout
- 桌面端优先：内容最大宽度 1200px，居中布局；两侧留白随屏幕增大而增加。
- 主要布局：页面采用“顶部导航 + 主内容容器 + 页脚（可选）”结构。
- 响应式策略：
  - ≥1200px：双栏/卡片栅格优先（例如结果页右侧为“可操作面板/生成报告”）。
  - 768–1199px：结果页双栏收敛为上下布局，卡片变为 1–2 列。
  - <768px（如后续支持）：所有模块单列堆叠，图表自适应宽度。

### Meta Information（建议）
- 默认 Title 模板：`{页面标题} - 简历分析与职业规划`
- 默认 Description：`上传简历，生成能力画像，匹配岗位并推荐职业路径，一键导出PDF报告。`
- Open Graph：
  - og:title = 页面标题
  - og:description = 同上
  - og:type = website

### Global Styles（设计 Token 建议）
- Colors
  - background: #0B1220（深色底）或 #F7F8FA（浅色底，二选一保持一致）
  - surface/card: #111827（深色）/ #FFFFFF（浅色）
  - primary: #4F46E5
  - success: #16A34A
  - warning: #F59E0B
  - danger: #EF4444
  - text primary: #0F172A（浅色底）/ #E5E7EB（深色底）
  - text secondary: #64748B（浅色底）/ #9CA3AF（深色底）
  - border: #E5E7EB（浅色底）/ #1F2937（深色底）
- Typography
  - H1 32/40, H2 24/32, H3 18/28
  - Body 14/22, Small 12/18
  - 等宽字体用于 ID/路径等技术字段（尽量少出现给用户）
- Components
  - Button：主按钮 primary 实心；次按钮 outline；危险操作使用 danger
  - Hover：按钮/卡片 hover 提升阴影或边框加深；链接 hover 下划线
  - Form：输入框聚焦边框 primary，错误态展示红色文案 + 输入框红边

### 通用组件（跨页复用）
- TopNav
  - 左侧 Logo + 产品名
  - 右侧：流程进度（上传/分析/报告）或简化为当前页标题
- Stepper（可选但建议）
  - 用于 3 步流程引导：上传解析 → 分析结果 → PDF 报告
- Toast/Alert
  - 展示上传失败、解析失败、PDF 生成失败等可重试信息
- Loading Skeleton
  - 能力画像/匹配结果/路径推荐加载时使用骨架屏

## Page 1：开始分析页（简历上传与解析）

### Meta
- title: 开始分析
- description: 上传简历并确认解析结果，开始生成能力画像与职业建议。

### Page Structure
- 上下堆叠的单页向导布局（桌面端内容居中），核心区域分为三段：上传区、解析预览区、偏好填写区。

### Sections & Components
1) 顶部区（Header）
- 标题：`上传简历并开始分析`
- 说明文案：支持 PDF/DOC/DOCX；隐私提示（简短）

2) 简历上传卡片（Upload Card）
- 组件：拖拽上传区（Dropzone）+ “选择文件”按钮
- 状态：
  - Idle：展示支持格式与大小提示
  - Uploading：进度条 + 文件名 + 取消按钮（可选）
  - Uploaded：显示文件名、大小、重新上传按钮
- 错误态：格式不支持/文件过大/网络失败的错误提示

3) 解析结果预览（Parse Preview）
- 组件：分组折叠面板（Accordion）
  - 基础信息：姓名/邮箱/电话/城市/工作年限（可编辑输入框）
  - 教育经历：表格或条目列表（可编辑）
  - 工作经历：卡片列表（公司、岗位、时间、要点；要点支持多行编辑）
  - 技能标签：可编辑 Tag 输入（支持新增/删除）
- 操作：
  - “重新解析”按钮（用于解析不完整时）
  - “继续”按钮在解析完成后可用

4) 分析偏好（Preferences）
- 组件：表单区域（两列栅格）
  - 目标岗位方向（必填，下拉 + 可自定义输入）
  - 目标行业（选填）
  - 目标城市（选填）
- 校验：目标岗位为空时禁止提交并提示

5) 主 CTA（Call to Action）
- 主按钮：`开始分析`（触发生成能力画像→匹配→路径推荐）
- 次按钮：`保存当前修改`（可选，若支持会话缓存）

## Page 2：分析结果页（能力画像→人岗匹配→职业路径）

### Meta
- title: 分析结果
- description: 查看你的能力画像、人岗匹配结果与职业路径推荐。

### Page Structure
- 桌面端推荐“双栏布局”：
  - 左侧（主内容，约 8/12）：能力画像 + 匹配结果 + 路径推荐
  - 右侧（操作面板，约 4/12）：目标与偏好调整、生成报告按钮、关键摘要

### Sections & Components
1) 顶部摘要条（Result Summary Bar）
- 内容：一句话总结（来自能力画像 summary）
- 标签：当前目标岗位方向、城市/行业偏好（可点击编辑）

2) 能力画像模块（Capability Profile）
- 卡片结构：
  - 左：雷达图（维度 5–8 个，清晰标注）
  - 右：优势 Top 3、短板/风险 Top 3（列表）
- 交互：
  - 点击维度弹出侧边抽屉（Drawer）：展示证据要点（evidence）与建议
- 状态：加载中显示骨架；失败显示“重试生成能力画像”

3) 人岗匹配模块（Job Matching）
- 列表：岗位卡片（按 matchScore 降序）
  - 卡片信息：岗位名、匹配度进度条、匹配理由（3条以内）
  - 展开后：差距清单（带优先级 P0/P1/P2）+ 对应补齐建议
- 交互：
  - 目标岗位方向改变后，按钮 `重新匹配`

4) 职业路径推荐模块（Career Path）
- 结构：三段式时间线（Short/Mid/Long）
  - 每段：目标岗位、预计时长（月）、里程碑、学习/项目清单（按优先级）
- 交互：
  - 支持切换“稳健路径/激进路径”（若后续扩展；当前可不展示）

5) 右侧操作面板（Action Panel）
- 表单：目标岗位方向、行业、城市（与结果联动）
- 主按钮：`生成PDF报告`
- 提示：生成报告将包含当前页面全部结果

## Page 3：报告预览与导出页（PDF）

### Meta
- title: 报告预览与导出
- description: 汇总分析结论并导出 PDF 报告。

### Page Structure
- 上方操作区 + 下方报告预览区（可滚动）。
- 桌面端预览区建议模拟 A4 页面比例（居中白底卡片），增强“报告感”。

### Sections & Components
1) 操作栏（Top Actions）
- 左：返回分析结果（链接/按钮）
- 右：
  - 主按钮：`生成PDF`
  - 次按钮：`下载PDF`（生成完成后可用）
  - 生成状态：loading/成功/失败提示

2) 报告预览（Report Preview）
- 章节结构（按顺序展示，可目录锚点可选）：
  - 封面：姓名（若有）、目标方向、生成日期
  - 个人概览：一句话定位 + 简要背景摘要
  - 能力画像：雷达图缩略 + 维度表（维度、分数、等级、证据摘要）
  - 人岗匹配：Top N 岗位表格（岗位、匹配度、差距要点）
  - 职业路径：短/中/长期时间线摘要
  - 行动计划：按优先级列出 10 条以内可执行事项（P0 优先）
- 视觉：
  - 使用统一的章节标题样式（H2）与分隔线
  - 关键结论用浅色背景 Callout（例如 primary 10% 透明底）

3) 错误与重试（Error State）
- 若 PDF 生成失败：展示错误原因（短文案）+ `重试生成` 按钮

4) 无数据保护（Guard）
- 当你直接访问 /report 且缺少分析数据：提示“请先完成简历分析”，提供返回首页按钮
